  <action>
Run these steps in order using the `gh` CLI and standard shell commands. All
commands operate on the working directory `/home/researcher/Teaching/inictel/clase_modelo`.

STEP 1 — Verify gh authentication:
  gh auth status
If not authenticated, `gh auth login` with the account ollerenac.

STEP 2 — Create the public repository (skip if it already exists):
  gh repo create ollerenac/clase_modelo --public --description "Clase modelo 15 min: EternalBlue / MS17-010"

STEP 3 — Write index.html to the working directory root.
The file must be pure HTML (no Jekyll front matter, no build directives).
Required content:
  - DOCTYPE html declaration
  - lang="es" on the html element
  - charset UTF-8 meta
  - A title element containing exactly: Clase Modelo: EternalBlue / MS17-010
  - An h1 element containing exactly: Clase Modelo: EternalBlue / MS17-010
  - A brief placeholder paragraph in Spanish indicating content is coming
    (e.g., "Contenido de la clase en construcción.")
  - Inline style block with at minimum: body { font-family: sans-serif; max-width: 800px; margin: 2rem auto; padding: 0 1rem; }

STEP 4 — Initialize git repo in the working directory (if not already a git repo),
set the default branch to main, add the remote pointing at ollerenac/clase_modelo,
commit index.html, and push:
  git init -b main                                   (skip if already a git repo)
  git remote add origin https://github.com/ollerenac/clase_modelo.git
  git add index.html
  git commit -m "feat(01): add skeleton index.html — walking skeleton"
  git push -u origin main

STEP 5 — Enable GitHub Pages via the GitHub API (source = root of main branch):
  gh api --method POST repos/ollerenac/clase_modelo/pages \
    --field source='{"branch":"main","path":"/"}' \
    2>/dev/null || \
  gh api --method PUT repos/ollerenac/clase_modelo/pages \
    --field source='{"branch":"main","path":"/"}'

The POST is used on first enable; the PUT updates an existing Pages config.
Either 201 or 200 response is a success. A 409 ("already enabled") is also fine.

STEP 6 — Confirm Pages is enabled:
  gh api repos/ollerenac/clase_modelo/pages --jq '.html_url'
Expected output: https://ollerenac.github.io/clase_modelo/

Do NOT add a .nojekyll file — the project constraint is pure HTML with no Jekyll,
so the default GitHub Pages behavior is correct (no Jekyll processing needed for
a plain .html file, since Jekyll only processes files with front matter).
